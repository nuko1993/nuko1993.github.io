内存对齐

  • 内存对齐主要是为了提高CPU寻址性能
  • 可以通过#pragma pack(n)来修改，其中n=1,2,4,8,16,n就是对齐系数
  • 内存对齐规则：
    • 数据成员对齐：struct或union的第一个成员在偏移为0的位置，以后每个数据的成员位置为min(对齐系数，自身长度)的整数倍
    • 数据成员为struct：以该struct内的最大长度的整数倍的位置开始存储
    • 整体对齐规则：按照1、2对齐后，自身大小也要对齐，对齐原则是min(对齐系数,数据成员最大长度)的整数倍

OC类的内存分析

相关函数

class_getInstanceSize
上面说到class_getInstanceSize表示成员变量占用的内存大小，那么对于Person类创建的实例来说，其成员变量应该是12，为什么结果是16.

class_getInstanceSize的内存也有自己的内存对齐，通过源码可知，其实现依赖于底层函数word_align，该函数返回的结果是8的倍数。另外从alloc函数开始分析，到class_getInstanceSize可以知道oc对象的内存大小至少是16字节，所以oc对象申请的内存空间是8字节对齐，且至少是16字节。

总结

  • class_getInstance:获取成员变量占用的内存大小
  • malloc_size:实际分配的内存大小
  • sizeof：传入变量或类型的大小
